## Project Identity

G1_Hand is an embedded firmware project for a **robotic dexterous hand controller** running on the **HPMicro HPM6E80** RISC-V MCU (`rv32imac` with `Zba_Zbb_Zbc_Zbs` extensions). It targets the `hpm6e00evk` evaluation board and builds against **HPM SDK v1.10.0**.

## Build System

- **Build tool**: CMake (≥3.13) with Ninja generator
- **Toolchain**: `riscv32-unknown-elf-gcc` (bundled with the SDK environment)
- **SDK environment**: `sdk_env_v1.10.0` at `C:/Users/Administrator/Downloads/sdk_env_v1.10.0/`

### Building

The environment variable `HPM_SDK_BASE` must point to the SDK root:

```sh
export HPM_SDK_BASE="C:/Users/Administrator/Downloads/sdk_env_v1.10.0/hpm_sdk"
```

Configure and build:

```sh
mkdir -p build && cd build
cmake .. -G Ninja -DBOARD=hpm6e00evk -DHPM_BUILD_TYPE=flash_xip
ninja
```

The existing build directory is `G1_Hand_hpm6e00evk_flash_xip_debug/`.

### Key build details

- Linker script: `flash_xip.ld` (execute-in-place from flash, 16 MB flash, 32 MB external RAM)
- Stack/heap: 16 KB each
- C standard flags: `-g -Wall -Wundef -Wno-format -fomit-frame-pointer -fno-builtin -ffunction-sections -fdata-sections`
- IDE projects for IAR EW and Segger Embedded Studio are auto-generated by `generate_ide_projects()` in CMakeLists.txt

## Architecture

The project follows a layered embedded firmware structure:

```
CMakeLists.txt              — 仅注册入口文件，IDE 项目由 generate_ide_projects() 自动生成
user_app/                   — 应用层
  app_main.c                — main() 入口
  tasks/                    — 任务模块
drivers/
  bsp/                      — 板级支持包（GPIO、UART HAL 封装）
  device_driver/            — 设备驱动（RS-485、电机 IC）
device/motor/               — 电机控制设备层代码
middlewares/utils/          — 中间件（kfifo 环形缓冲区）
```

**添加新文件时无需修改 CMakeLists.txt**。IDE 项目文件（Segger Embedded Studio / IAR EW）会自动发现并注册新的 `.c`/`.h` 源文件，`CMakeLists.txt` 仅保留入口文件 `user_app/app_main.c`，其余由 IDE 生成器处理。

The SDK provides all MCU peripheral drivers (`hpm_uart_drv`, `hpm_dma_drv`, `hpm_dmamux_drv`, `hpm_clock_drv`, `hpm_l1c_drv`, etc.) and board-level initialization via `board.h`.

## Current State

This is an **early-stage project**. The only functional code is [app_main.c](user_app/app_main.c), which is a UART DMA echo demo (receive 16 bytes via DMA, echo them back). All files in `drivers/bsp/` and `drivers/device_driver/` are empty stubs (only `#include` guards). The `device/` and `middlewares/` directories are empty.

The build is configured and compiles successfully against the HPM SDK. The next development step is implementing the motor control drivers and RS-485 communication layer.
